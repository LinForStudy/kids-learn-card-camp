# 星卡指挥营 UI 二次重构提示词

你是资深前端工程师 + 游戏 UI 设计师。请基于当前 kids-learn-card-camp 项目，按照我提供的 6 张参考图重构 UI。图片只作为布局、层级、风格、组件比例参考，不要把图片直接当素材塞进页面，不要照抄图片中的错误文字，不要复制任何 KARDS 官方素材、Logo、卡图、卡名或完整规则。

参考图说明：
1. 01_base_home_reference.png：基地页方向。用于主基地、成长循环、补给箱状态、当前卡组预览、底部导航。
2. 02_training_learning_reference.png：训练页方向。学习模式要明亮、清爽、低压力，和战场深色风格分开。
3. 03_card_library_reference.png：卡库页方向。用于补给箱、收藏册、当前卡组、两列卡牌网格、费用曲线。
4. 04_battle_lobby_reference.png：战场入口方向。用于战场入口页，不代表战斗进行页。
5. 05_battle_playing_reference.png：战斗进行页方向。最重要。战斗中必须隐藏顶部品牌大横幅和底部导航，做成全屏卡牌战场。
6. 06_card_template_spec_reference.png：卡牌组件规范。用于完整卡牌、小型战场单位卡、手牌卡、卡组列表条、稀有度边框、阵营标识。

核心要求：
- 学习要像学习 App：浅色、清楚、温和、大按钮、答错不羞辱。
- 玩要像真实卡牌游戏：深绿黑金、战场桌面、总部、前线、支援区、手牌、操作面板。
- 战场模块拆成 battle-lobby 和 battle-playing。
- battle-lobby 可以显示底部导航。
- battle-playing 必须隐藏底部导航，隐藏顶部品牌大横幅。
- battle-playing 尽量单屏显示：敌方 HUD、敌方支援区、敌方前线、前线控制条、我方前线、我方支援区、我方 HUD、手牌区、操作面板。
- 不要做成长滚动信息流。
- 空槽不要像表格，降低虚线存在感；单位卡、手牌卡、选中状态要更突出。
- 手牌底部横向滑动；点击手牌后出现半高详情操作面板。
- 将 Kredit 全部改成“行动点”。
- 卡库页使用“补给箱 / 收藏册 / 当前卡组”三个 Tab。
- 收藏册使用两列卡牌网格。
- 卡牌组件必须包含：费用、阵营、卡名、插画区、技能、攻击、生命、稀有度、拥有数量。
- 训练页使用浅色学习风格，不使用战场深绿色压迫感。

实现约束：
- 只修改 kids-learn-card-camp/，不要动 kids-english-play-app/。
- 保持原生 HTML/CSS/JS，不引入 React/Vue。
- 保持 UTF-8 编码。
- 不联网、不排行榜、不社交、不广告、不付费抽卡。
- 不复制 KARDS 官方素材、Logo、卡图、卡名、完整数据库。

建议执行顺序：
1. 先整理 CSS 变量和通用组件：game-shell、status-pill、primary-action、section-panel、card-frame、battle-slot、bottom-nav。
2. 重构基地页和训练页。
3. 重构卡库页和卡牌组件。
4. 拆分 battle-lobby / battle-playing。
5. 重点重做 battle-playing 全屏战场。
6. 运行 node --check 检查所有 JS 文件，修复错误。

验收标准：
1. 战斗进行页看不到底部导航。
2. 战斗进行页看不到顶部“星卡指挥营”大品牌介绍。
3. 战斗进行页第一屏能看到敌方区域、前线控制、我方区域、手牌和操作面板。
4. 训练页明显是浅色学习风格。
5. 卡库页有补给箱、收藏册、当前卡组三个 Tab。
6. 收藏册是两列卡牌网格。
7. 卡牌有完整实体感。
8. 手牌横向滑动，选中后可部署/推进/攻击。
9. 刷新不丢失本地存档。
10. 所有 JS 语法检查通过。
